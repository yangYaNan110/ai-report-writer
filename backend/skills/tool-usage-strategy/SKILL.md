# Tool Usage Strategy - 工具调用策略

## 核心原则
在适当的时候调用工具，不滥用工具，工具调用要精准高效。

## 可用工具说明

### 1. 搜索工具 (web_search)
- **用途**：获取最新信息、查找数据、验证事实
- **调用时机**：
  - 用户明确要求查最新/实时数据
  - 需要最近1-2年的数据（当前时间之后的信息）
  - 需要具体的数据支撑
  - 不确定的事实需要验证
  - 用户问题涉及当前趋势、最新发展

### 2. 文件解析工具 (parse_document)
- **用途**：解析用户上传的PDF、DOCX等文件
- **调用时机**：
  - 用户上传了文件
  - 需要从文件中提取信息
  - 需要分析文件内容

### 3. 内容分析工具 (analyze_content)
- **用途**：深度分析文本内容
- **调用时机**：
  - 需要对长文本做摘要
  - 需要提取关键词
  - 需要情感分析
  - 需要对搜索结果进行整理分析

## 工具调用优先级

1. **文件优先**：如果用户上传了文件，优先解析文件
2. **数据优先**：需要数据时，优先用搜索获取最新数据
3. **本地优先**：能用已有知识解决的，不调用工具
4. **实时优先**：涉及时间敏感信息，必须用搜索

## 时间敏感性判断

- **过去的事件/历史知识**：可用模型知识，不需要搜索
- **当前趋势/最新数据**：必须搜索
- **未来预测**：基于当前数据+趋势分析，可能需要搜索获取基础数据

## 组合使用策略

当需要写一篇数据报告时：
1. 先用搜索获取最新数据
2. 然后用分析工具整理数据
3. 最后结合写作规范输出报告

## 示例场景

用户问："帮我写一份AI芯片市场分析报告"

思考过程：
1. 需要最新市场数据 → 调用 web_search 搜索"AI芯片市场 最新报告 2024"（注：当前年份动态替换）
2. 获取数据后 → 整理成报告格式
3. 不需要其他工具 → 直接写作

用户问："分析一下我上传的这份财报"

思考过程：
1. 用户上传了文件 → 调用 parse_document 解析文件
2. 获取内容后 → 调用 analyze_content 分析关键指标
3. 结合分析结果 → 输出分析报告
